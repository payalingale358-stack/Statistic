08

Measures of Position

Where does a single value stand within a distribution?

8.1

Z-Scores (Standardization)

📐
// What is it?

A Z-score tells you how many standard deviations a data point is from the mean. It removes units and puts all data on a common scale — essential for comparing apples to oranges in data science.

// Formula
Z = (X − μ) / σ
X = individual value  |  μ = mean  |  σ = standard deviation
// Step-by-Step Process
1

Find the mean (μ) of the dataset.

2

Find the standard deviation (σ) of the dataset.

3

Subtract the mean from the data point: (X − μ)

4

Divide that result by the standard deviation.

5

Interpret: Z=0 → at mean | Z=+2 → 2 SDs above | Z=−1 → 1 SD below.

// Practical Example — Exam Scores
Dataset: Student scores → [55, 60, 65, 70, 75, 80, 85, 90, 95, 100]
Mean (μ) = (55+60+65+70+75+80+85+90+95+100) / 10 = 775/10 = 77.5
σ = √[(Σ(X−μ)²)/n] ≈ 14.36
Z for X=55 → (55 − 77.5) / 14.36 = −22.5 / 14.36 ≈ −1.57
Z for X=100 → (100 − 77.5) / 14.36 = 22.5 / 14.36 ≈ +1.57
✓ Score of 55 is 1.57 SDs below average. Score of 100 is 1.57 SDs above.
Z-scores > 3 or < −3 are typically flagged as outliers in data science pipelines. Libraries like scikit-learn use Z-score standardization (StandardScaler) before feeding data to ML models.
8.2

Percentile Ranks

📊
// What is it?

A percentile rank tells you the percentage of values in a dataset that fall at or below a given value. If you're in the 90th percentile, you scored better than 90% of the group.

// Formula
PR = (Number of values ≤ X / Total N) × 100
Alternative: PR = (L + 0.5E) / N × 100   where L = below X, E = equal to X
// Step-by-Step Process
1

Sort the dataset in ascending order.

2

Count how many values are below the target value (L).

3

Count how many values are equal to the target (E).

4

Apply formula: (L + 0.5E) / N × 100

// Practical Example — Test Scores
Sorted scores: [42, 55, 58, 63, 67, 72, 75, 80, 88, 95] → N = 10

Find the percentile rank of score = 72:

L (values below 72) = 5 (42, 55, 58, 63, 67)
E (values equal 72) = 1
PR = (5 + 0.5×1) / 10 × 100 = (5.5 / 10) × 100 = 55
✓ Score of 72 is at the 55th percentile — better than 55% of students.
NumPy's np.percentile() and pandas' .quantile() use this concept heavily for data exploration and outlier detection in EDA (Exploratory Data Analysis).
8.3

Deciles

🔟
// What is it?

Deciles divide sorted data into 10 equal parts. D1 = 10th percentile, D5 = median (50th percentile), D9 = 90th percentile. They are a coarser but cleaner way to describe distribution spread.

// Formula (for grouped data)
Dₙ = L + [ (nN/10 − CF) / f ] × h
L = lower boundary | N = total | CF = cumulative freq before class | f = class freq | h = class width
// Step-by-Step (Ungrouped Data)
1

Sort the data in ascending order.

2

Calculate the position: Position = (n × N) / 10

3

If position is whole → average that value and the next.

4

If position is decimal → round up, take that indexed value.

// Practical Example — Monthly Salaries (₹ thousands)
Sorted: [18, 22, 25, 30, 34, 38, 42, 47, 55, 60] → N = 10
DecilePosition FormulaPositionValue (₹k)Meaning
D11×10/10118Bottom 10% earn ≤ ₹18k
D33×10/10325Bottom 30% earn ≤ ₹25k
D55×10/105.5 → avg(34,38)36Median salary = ₹36k
D99×10/10955Top 10% earn ≥ ₹55k
✓ D9 − D1 = 55 − 18 = ₹37k → This is the interdecile range (spread of middle 80%).
In data science, deciles are used in credit scoring (risk buckets), A/B test analysis, and feature binning (pd.qcut with q=10 in pandas).
8.4

Quartile Deviation

📦
// What is it?

Quartile Deviation (QD), also called the Semi-Interquartile Range, measures the spread of the middle 50% of data. It is half the distance between Q3 (75th percentile) and Q1 (25th percentile). It's robust to outliers — unlike standard deviation.

// Formulas
Q1 = value at (N+1)/4 th position
Q3 = value at 3(N+1)/4 th position
IQR = Q3 − Q1
QD = (Q3 − Q1) / 2
Coefficient of QD = (Q3 − Q1) / (Q3 + Q1)
// Step-by-Step
1

Sort data in ascending order.

2

Find Q1: value at position (N+1)/4

3

Find Q3: value at position 3(N+1)/4

4

Calculate QD = (Q3 − Q1) / 2

5

For comparison across datasets → use Coefficient of QD.

// Practical Example — House Prices (₹ Lakhs)
Sorted: [15, 20, 25, 30, 35, 40, 45, 50, 55, 60, 65] → N = 11
Q1 position = (11+1)/4 = 3rd value → Q1 = 25
Q3 position = 3×(11+1)/4 = 9th value → Q3 = 55
IQR = Q3 − Q1 = 55 − 25 = 30
QD = 30 / 2 = 15
Coefficient of QD = (55−25)/(55+25) = 30/80 = 0.375
✓ QD = ₹15 Lakhs. Middle 50% of houses are within ±₹15L of median.
In pandas: df['price'].quantile(0.75) gives Q3. QD is the backbone of box plots — Matplotlib and Seaborn use Q1, Q3, and IQR to draw whiskers. Values beyond 1.5×IQR are plotted as outlier dots.
8.5

Standard Scores

📏
// What is it?

Standard scores are a family of transformed scores that convert raw data into a common interpretable scale. Z-score is one type. Others include T-scores (mean=50, SD=10) and Stanine scores (1–9 scale). They make cross-dataset comparison possible.

// Standard Score Transformations
Z-score : Z = (X − μ) / σ [mean=0, SD=1]
T-score : T = 50 + 10Z [mean=50, SD=10]
Stanine : S = INT(2Z + 5.5) [scale 1-9, mean≈5]
IQ Scale : IQ = 100 + 15Z [mean=100, SD=15]
// Practical Example — Comparing Two Students Across Subjects
Arjun: Math=80 (μ=70, σ=10) | English=75 (μ=65, σ=15)
Priya: Math=85 (μ=70, σ=10) | English=70 (μ=65, σ=15)
StudentSubjectRaw ScoreZ-ScoreT-ScoreBetter?
ArjunMath80+1.060—
ArjunEnglish75+0.6756.7—
PriyaMath85+1.565✓ Math
PriyaEnglish70+0.3353.3—
✓ Priya is better in Math (Z=+1.5 vs +1.0) but Arjun performs better in English relative to his peers (Z=+0.67 vs +0.33).
Data science use case: scikit-learn's StandardScaler produces Z-scores. MinMaxScaler and RobustScaler are other standard score transformations used to normalize features before training ML models.
09

Measures of Relationship

How do two variables move together — and how strongly?

9.1

Covariance

🔗
// What is it?

Covariance measures the direction of the linear relationship between two variables. A positive covariance means both increase together. Negative means one increases as the other decreases. Zero means no linear relationship. However, the magnitude is hard to interpret because it depends on the units.

// Formula
Cov(X,Y) = Σ[(Xᵢ − X̄)(Yᵢ − Ȳ)] / N
For sample: divide by (N−1) instead of N (Bessel's correction)
// Step-by-Step
1

Calculate the mean of X (X̄) and mean of Y (Ȳ).

2

For each pair (Xᵢ, Yᵢ), compute (Xᵢ − X̄) and (Yᵢ − Ȳ).

3

Multiply these deviations: (Xᵢ − X̄)(Yᵢ − Ȳ)

4

Sum all products, divide by N (population) or N−1 (sample).

5

Interpret sign: positive (+) = same direction | negative (−) = opposite direction.

// Practical Example — Study Hours vs. Exam Score
X (Hours): [2, 4, 5, 6, 8] | Y (Score): [50, 60, 65, 70, 80]
X̄ = (2+4+5+6+8)/5 = 25/5 = 5
Ȳ = (50+60+65+70+80)/5 = 325/5 = 65
XYX−X̄Y−Ȳ(X−X̄)(Y−Ȳ)

Sum of products = 45+5+0+5+45 = 100
Cov(X,Y) = 100 / 5 = 20
✓ Cov = +20 → Positive relationship: more study hours → higher scores. But we can't compare this 20 with anything — we need correlation for that.
In Python: np.cov(X, Y) returns a 2×2 covariance matrix. The off-diagonal values are the covariance between X and Y. The covariance matrix is central to PCA (Principal Component Analysis).
9.2

Coefficient of Correlation (Pearson's r)

📈
// What is it?

Pearson's correlation coefficient (r) is the standardized covariance — it tells you both the direction and strength of a linear relationship between two variables. It always lies between −1 and +1, making it interpretable regardless of units.

// Formula
r = Cov(X,Y) / (σₓ × σᵧ)
= Σ[(Xᵢ−X̄)(Yᵢ−Ȳ)] / √[Σ(Xᵢ−X̄)² × Σ(Yᵢ−Ȳ)²]
r = +1 → perfect positive | r = 0 → no linear relationship | r = −1 → perfect negative
// Interpretation Scale
r

0.90 to 1.00 → Very Strong positive correlation

r

0.70 to 0.89 → Strong positive correlation

r

0.50 to 0.69 → Moderate positive correlation

r

0.00 to 0.49 → Weak or no correlation

// Practical Example — Study Hours vs. Exam Score (continued)
Using same data: X=[2,4,5,6,8], Y=[50,60,65,70,80]
Cov(X,Y) = 20 (calculated above)
Σ(X−X̄)² = (−3)²+(−1)²+0²+1²+3² = 9+1+0+1+9 = 20 → σₓ = √(20/5) = 2
Σ(Y−Ȳ)² = 225+25+0+25+225 = 500 → σᵧ = √(500/5) = 10
r = Cov(X,Y) / (σₓ × σᵧ) = 20 / (2×10) = 20/20 = 1.0
✓ r = +1.0 → Perfect positive linear correlation. Every extra study hour perfectly predicts the score increase.
// Real-world Example — Ice Cream vs. Drowning Rate
Month: [Jan, Feb, Mar...Aug] | Ice Cream Sales: [100, 110, 130...400] | Drowning: [5, 6, 7...25]
Suppose r = +0.85 → Strong positive correlation
⚠ Correlation ≠ Causation! Both are driven by the hidden variable: Summer Heat. This is called a spurious correlation — critical concept in data science.
Python: scipy.stats.pearsonr(X, Y) returns (r, p_value). The p-value tells you if the correlation is statistically significant. df.corr() in pandas computes pairwise Pearson correlation matrix for all numeric columns.
9.3

Scatter Plots Interpretation

🌐
// What is it?

A scatter plot is a 2D graph that plots each observation as a point using its (X, Y) values. It's the first visual tool to assess whether two variables have a linear, non-linear, or no relationship. Every data scientist starts correlation analysis by looking at the scatter plot.

Pattern Types
STRONG POSITIVE
r ≈ +0.95
STRONG NEGATIVE
r ≈ −0.92
NO CORRELATION
r ≈ 0.00
// How to Interpret a Scatter Plot — Checklist
1

Direction: Do points move up-right (positive) or down-right (negative)?

2

Strength: How tightly clustered are points around an imaginary line? Tight = strong, scattered = weak.

3

Form: Is the pattern linear (straight line) or curved (non-linear)? Pearson r only measures linear relationships!

4

Outliers: Are there any points far away from the main cluster? These can distort r significantly.

5

Clusters: Are there sub-groups? Multiple clusters may indicate hidden categorical variables.

// Practical Example — Height vs. Weight
X (Height in cm): [155, 160, 165, 170, 175, 180, 185] | Y (Weight in kg): [50, 55, 60, 65, 70, 75, 80]
What we observeInterpretation
Points go up-rightPositive direction → Taller people tend to weigh more
Points tightly clusteredStrong relationship → r ≈ +0.99
Straight line patternLinear form → Linear regression is appropriate
No extreme outliersData is clean, r is reliable
✓ A high, positive, linear scatter tells us to use Pearson r and linear regression to model this relationship.
// Warning — Anscombe's Quartet

Four datasets with identical summary statistics (mean, variance, r) but completely different scatter patterns:

Datasetr valueVisual PatternAppropriate Model?

⚠ ALWAYS visualize your data before trusting correlation numbers. This is the #1 rule in EDA.
Python code to create scatter plots: matplotlib.pyplot.scatter(X, Y) or seaborn.scatterplot(x='col1', y='col2', data=df). For full pair-wise scatter matrix: seaborn.pairplot(df) — a staple in every data science EDA workflow.
